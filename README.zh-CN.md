# zettelkasten-graph

> 卡片盒笔记 + Agent + 图论 + LLM 蒸馏。人机共构的知识库。

AI 帮你链接笔记。图论帮你审计 AI。LLM 将其万亿参数的判断蒸馏到你的个人知识权重中。把笔记变成社交网络——排名、导航、发现隐藏的知识聚类。

## 核心洞察 — 从大公司的万亿参数中蒸馏个人知识

你下载不了大厂最强模型的权重。但你可以**提取**它的知识。

每一条 Zettelkasten 笔记编码了一个理解。每一条加权链接编码了一个判断——两个概念有多近。这和神经网络的注意力机制完全同构——概念通过加权边相连。

**zettelkasten-graph** 将这种相似性转化为一条管线：

1. 你写笔记 + 链接（你的个人知识图）
2. LLM 阅读每个节点的出链，重新分配注意力 → LLM 成为你的**教师模型**
3. 对比：你的权重 vs LLM 的注意力分布 → 暴露偏差
4. 你决定：信任教师还是自己的直觉

这是**个体级别的知识蒸馏**。你写的每一条笔记，设定的每一条链接，都是从耗资万亿训练的模型中蒸馏出来的。你的图每跑一次就更好一点。明天出了更强的模型——重跑管线。图不变，教师升级。

```
# 蒸馏单个节点
python scripts/distill_attention.py "笔记名"

# 试运行（只看 prompt，不调 API）
python scripts/distill_attention.py "笔记名" --dry-run
```

## 解决的问题

传统 Zettelkasten 用文件夹、MOC 和平面的 `[[链接]]`。这导致：
- 文件夹强加了人为层级
- MOC 成为"超级枢纽"，扭曲网络指标
- 无法量化哪些笔记真正重要
- 无法检测知识空白或过载概念
- 无法验证你设的链接权重是否正确

## 解决方案 — 14 种图分析方法

| # | 方法 | 做什么 |
|:---|:---|:---|
| 1 | **核心排序** (PageRank) | 哪些笔记全局最重要 |
| 2 | **桥梁检测** (介数中心性) | 哪些笔记连接不同知识域 |
| 3 | **自然分组** (Louvain) | 知识如何自然聚成主题 |
| 4 | **概念路径** (Dijkstra) | 任意两个概念之间的最优路径 |
| 5 | **补链推荐** (Jaccard/Adamic-Adar) | 哪些笔记应该连接 |
| 6 | **链接质量审查** | 检测不对称、传递缺失、权重偏差 |
| 7 | **拆分建议** (过载检测) | 哪个笔记承载了过多概念 |
| 8 | **空白定位** (缺失节点预测) | 哪里需要一个新概念 |
| 9 | **角色区分** (HITS) | Hub（导航者）vs Authority（基石） |
| 10 | **知识内核** (K-core) | 剥掉表层后剩下的硬核概念 |
| 11 | **最佳起点** (紧密度) | 从哪里出发最快到达全部知识 |
| 12 | **区域密度** (聚类系数) | 哪些知识区域链接扎实 |
| 13 | **断点检测** (割点) | 移除哪些节点图会断裂 |
| 14 | **知识宽度** (直径) | 你的知识网络有多宽广 |

## 链接语法

完全兼容 Obsidian 的 `[[wikilink]]` 语法，在原生基础上扩展了权重标注。

```markdown
[[目标]]                 → 默认权重 0.5，有向边 A→B
[[目标]] 0.9             → 权重 0.9（亲密度，0.01–1.0）
[[目标]] 0.3 — 描述      → 权重 + 描述
```

方向：在笔记 A 中写 `[[B]]` = A→B。若 B 也写了 `[[A]]`，则形成双向边。

## 使用方法

### 安装

```bash
pip install -r requirements.txt
```

配置 `scripts/.env`，填入你的笔记目录路径和 API 密钥。

### 分析

```bash
python scripts/knowledge_graph_analyzer.py
```

### 查询

```bash
python scripts/knowledge_graph_query.py 社区          # 列出所有社区
python scripts/knowledge_graph_query.py 核心 10       # PageRank 前 10
python scripts/knowledge_graph_query.py 邻居 笔记名    # 某个笔记的入链/出链
python scripts/knowledge_graph_query.py 路径 A B      # 两概念间最短路径
python scripts/knowledge_graph_query.py 扩展 笔记名 2  # BFS 2 跳
python scripts/knowledge_graph_query.py 桥接          # 介数中心性桥接笔记
```

### 蒸馏

```bash
python scripts/distill_attention.py "笔记名"
```

## 原则

1. **不要文件夹。** 所有笔记扁平放在一个目录。图是唯一的组织结构。
2. **不要 MOC。** 索引文件是人造的超级枢纽，会扭曲指标。
3. **每条链接加权。** 不是所有连接都一样。使用 0.01–1.0。
4. **图驱动置信度。** 低自信 + 高图验证 → 调高置信度。
5. **零幽灵链接。** 绝不链接不存在的笔记。待建笔记用 `## 待探索` 纯文本记录。
