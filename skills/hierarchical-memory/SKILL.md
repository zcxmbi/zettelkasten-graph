# 层次记忆管理

> **v2.4**：补充 `memory` tool 作写入口径。MEMORY.md 为 memory tool 持久化的产物，不再手动编辑。新增"外部内容→隐藏记忆"管道流程（知新管道）。

## 一、三层记忆模型

| 层级 | 存储位置 | 写入口径 | 上下文可见性 | 晋升条件 |
|------|---------|---------|-------------|---------|
| **活跃记忆** | `memory` tool 存储 / MEMORY.md 产物 | `memory(action='add'/'replace'/'remove', target='memory')` | 每轮对话自动加载 | 参考记忆使用 ≥10 次 |
| **参考记忆** | `memory` tool 存储 / MEMORY.md 产物 | 同上，内容以 `[参考]` 开头 | 按需检索 | — |
| **隐藏记忆** | `D:\\文件\\我的知识体系\\` 中的 `.md` 文件 | `write_file` 写入 `.md`，然后 `memory(action='add')` 登记 `[参考]` 行 | 不加载，不占上下文 | 被动引用、主动检索或记忆登记命中 1 次 |

**特殊规则：实践优先原则**

> **实践是检验真理的唯一标准。**
> 任何记忆（无论层级）一旦与实践产生冲突，必须修改该记忆以反映实践结果。

## 二、物理存储

### 2.1 知识体系文件夹 — 隐藏记忆池（冷存储）
- 路径：`D:\文件\我的知识体系`
- 所有 `.md` 文件天然是隐藏记忆
- 不占上下文，通过搜索唤醒
- **Git 版本控制**

### 2.2 memory tool 与 MEMORY.md 的关系

- **写入口径**：始终通过 `memory(action='add'/'replace'/'remove', target='memory')` 操作，**不手动编辑 MEMORY.md**
- MEMORY.md 是 memory tool 自动持久化的产物文件，格式（`§`分隔 + `[参考]`行）由 tool 维护
- 三层模型中的"隐藏记忆→参考记忆→活跃记忆"的晋升退化，通过恰当选择 `target=` 和操作类型实现

### 2.2.1 项目文档 vs 通用知识 存放规则

**关键区分**（来自用户纠正）：

| 内容类型 | 存放位置 | 登记方式 |
|:---------|:---------|:---------|
| **通用知识**（可复用于多个项目的概念、理论、模板） | `D:\文件\我的知识体系\` | `memory(action='add')` 加 `[参考]` 行 |
| **项目设计文档**（专属于某个游戏/项目的设计文件） | `D:\Game\Project\<项目名>\` | `memory(action='add')` 加 `[参考]` 行，路径指向项目文件夹 |
| **项目内的验证脚本/工具** | `D:\Game\Project\<项目名>\scripts\` | 不登记 reference |

**不把项目文档放入 `D:\文件\我的知识体系\`**，即使主题相同。文件属于项目则放在项目里，知识体系只存通用化的、跨项目可复用的抽象知识。

当同时操作两类内容时，先确认目标文件夹再写入，避免写错位置后二次搬运。

### 2.3 知新管道：外部内容 → 隐藏记忆

将学习成果持久化的标准流程：

1. **读取** — `read_file` 获取外部内容（视频文案、文章、论文等）
2. **提取** — 从内容中抽取出目标主题的所有相关论述
3. **补充** — 用自己的知识补充外部来源未覆盖的内容（标记为"补充"）
4. **结构化** — 按 Zettelkasten 格式写成 `.md` 笔记文件，存入 `D:\文件\我的知识体系\`
5. **登记** — `memory(action='add', target='memory')` 以 `[参考]` 格式登记，供后续按需检索
6. **索引** — 若为新领域，创建 `_索引.md` MOC 笔记

注意：网络不可用的 fallback 模式下，使用自身知识储备替代 web 搜索，并注明哪些是原文引用、哪些是补充推导。

### 2.3 数据流动

## 三、记忆写入规范

### 活跃记忆（通过 `memory` tool）

```python
# 单条写入
memory(action='add', target='memory', content='代理软件 Shark Cloud，HTTP 代理端口 7897…')

# 批量原子写入（推荐：先清旧后加新，防超限）
memory(action='add', target='memory',
       operations=[{'action': 'remove', 'old_text': '旧条目关键字'},
                    {'action': 'add', 'content': '新条目内容'}])

# 替换/更新
memory(action='replace', target='memory',
       old_text='旧条目标识子串', content='新内容')

# 删除
memory(action='remove', target='memory', old_text='要删除条目标识子串')
```

### 参考记忆格式

内容以 `[参考]` 开头：

```
[参考] 核心文件.md | count=2 | reference\核心文件.md | 查询结构时
```

- memory tool 自动持久化到 `§` 分隔的存储
- 不手动编辑产物文件
- 路径用相对路径（`reference\\xxx.md`），避免长绝对路径折断

### 隐藏记忆写入（知识体系）

```python
# 写入笔记
write_file(path='D:\\文件\\我的知识体系\\游戏设计\\双被论锦标赛_7节点模型.md',
           content='# 标题\n\n正文…')

# 登记参考记忆
memory(action='add', target='memory',
       content='[参考] 双被论锦标赛 | count=0 | 游戏设计\\双被论锦标赛_7节点模型.md | 7节点2-King Tournament模型')
```

## 四、晋升与退化规则

### 4.1 旧规则（已废弃）

> ~~参考记忆使用 ≥10 次 → 活跃记忆。~~ 已替换为图驱动规则。

### 4.2 图驱动晋升规则

晋升不再依赖机械的使用计数，改为知识图网络的结构验证：

**晋升条件（隐藏/参考 → 活跃记忆）**：
- 入度 ≥ 7，且被 ≥ 3 个社区引用，且 PageRank ≥ 0.01
- 满足条件的笔记是"从图网络中自然涌现的核心节点"——被多个领域、多种类型的笔记引用，不是人为标记的

**置信度调整**：
- 入度 ≥ 8 且跨 ≥ 3 社区 → 置信度应 ≥ 80%
- 入度 ≥ 5 且跨 ≥ 2 社区 → 置信度应 ≥ 60%
- PageRank ≥ 0.015 → 置信度应 ≥ 50%
- **图验证高于主观判断**：如果 confidence < 图验证建议值，将 confidence 调至建议值

**Agent 规则**：主动汇报晋升候选和置信度偏低的笔记，但**不自动执行**——需用户确认。

### 4.3 退化：活跃记忆 → 参考记忆
- 触发：stale_tokens > 80,000
- 操作：写回知识体系文件夹（若无），活跃条目降为 `[参考]` 行

## 八、置信度

| 置信度 | 含义 | 冲突时 |
|--------|------|--------|
| 90-100% | 大量实践验证 | 优先审查自己的理解 |
| 70-89% | 已验证的个人方法 | 平等对待 |
| 40-69% | 个人推导 | 倾向修改知识 |
| 10-39% | 灵感/假设 | 优先修改知识 |

## 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 手动编辑 MEMORY.md / 产物文件 | 与 `memory` tool 写冲突，内容被下一轮覆盖 | 始终通过 `memory(action=...)` 操作 |
| 忘记登记隐藏记忆（只写文件不调 memory） | 参考链断裂，无法按需检索 | 写入知识体系文件后立即 `memory(action='add')` 加 `[参考]` 行 |
| 使用绝对路径（超长） | 行被折断，正则失效 | 用相对路径 |
| 把规则说明写入记忆内容 | 数据文件被污染 | 规则写 skill / SOUL.md |
