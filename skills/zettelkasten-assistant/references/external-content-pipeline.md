# 外部内容 → 结构化笔记管道（知新管道）

> 处理视频文案、文章、论文等外部内容，提取核心知识存入隐藏记忆。

## 流程

1. **读取** — `read_file` 获取外部内容
2. **通读标注** — 扫描全文，标记与目标主题相关的段落（时间戳/行号）
3. **提取** — 从标记段落中逐条抽取原文论述，保留上下文
4. **补充** — 用自己的知识补充外部来源未覆盖的内容（明确标注哪些是原文、哪些是补充）
5. **结构化** — 按 Zettelkasten 格式写成 `.md` 笔记文件
   - 存到 `<vault-path>/<领域>\<文件名>.md`
   - YAML frontmatter：`tags`, `source`
   - 正文分节：核心定义、原文提取、补充知识、设计应用、相关链接
6. **登记** — `memory(action='add', target='memory', content='[参考] 名称 | count=0 | 路径 | 说明')`
7. **索引** — 若为新领域，创建该领域的 `_索引.md` MOC

## 格式模板（知识体系笔记）

```markdown
---
tags: [领域1, 领域2]
source: 来源名称
---

# 标题

> 一句话摘要。

## 一、核心定义

## 二、原文提取

（精确引用，含时间戳）

## 三、补充知识

（自己的知识 + 数学推导 + 未覆盖内容）

## 四、应用框架

## 五、注意事项

## 相关笔记

- [[相关笔记名]]
```

## 网络不可用的 fallback 模式

当 `web_search` 或浏览器不可用时：
- 使用自身知识储备替代网络检索
- 在笔记中标注哪些是原文引用、哪些是补充推导
- 可在"待探索"区块记录后续需要网络验证的内容
