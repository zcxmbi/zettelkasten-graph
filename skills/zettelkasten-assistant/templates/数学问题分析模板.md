## 使用规则

- **不写代码**：这是数学笔记，代码由 Claude Code 负责。只写公式、概念、取舍理由。
- **层级优先**：先想清楚这个问题可以拆成几层，每层独立的数学工具是什么。
- **多路线记录**：不只记最终方案——记录探索过的路线和淘汰理由。
- **可调参数**：数学方案中暴露给游戏的参数（如 σ₀、k、α）要明确标注，方便后续调参。
