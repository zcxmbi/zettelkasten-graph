# 图论知识分析方法

对 Zettelkasten 知识网络应用图论方法，量化评估链接质量、发现缺失连接、优化导航结构。

## 已实现方法（4个）

### PageRank — 全局重要性
- **原理**：随机游走 + 阻尼系数 0.85。被重要笔记引用的自己也重要。
- **公式**：$PR(A) = \frac{0.15}{N} + 0.85 \sum_{B \to A} \frac{PR(B)}{出度(B)} \cdot w_{B \to A}$
- **命令**：`python knowledge_graph_query.py 核心 20`
- **用途**：找到知识体系的基石概念，优先级高于入度排名。

### 介数中心性 — 桥接笔记
- **原理**：节点出现在多少条最短路径上。加权版用 Dijkstra。
- **公式**：$B(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}$
- **命令**：`python knowledge_graph_query.py 桥接`
- **用途**：找到结构性支柱——移除它们会割裂知识网络。优先复习和备份。

### Louvain 社区检测 — 自然聚类
- **原理**：贪心优化模块度 $Q$，让社区内部边多、社区间边少。
- **命令**：`python knowledge_graph_query.py 社区`
- **用途**：发现链接权重自然形成的聚类。与主观分类对比，揭露隐藏关联。

### Dijkstra 加权最短路径 — 语义距离
- **原理**：距离 = 1/权重。高权重边 = 高速公路，低权重边 = 泥巴路。
- **命令**：`python knowledge_graph_query.py 路径 A B`
- **用途**：量化概念间的语义距离。路径长的概念对需要补高权重链接。

## 待实现方法（7个）

### HITS（Hub/Authority）
- 区分"路标型"笔记（出链多）和"基石型"笔记（入链多）。

### K-core 分解
- 像剥洋葱一样迭代移除低度节点。最内核 = 知识的"硬核"。

### 紧密度中心性
- 从该节点出发到达所有其他节点的平均距离的倒数。最好的"探索起点"。

### 聚类系数
- 邻居之间互相连接的程度。局部密度指标，找出"稀疏区域"补链接。

### 割点检测
- 移除后图连通分量增加的节点。二元版介数——"不可遗忘"清单。

### 最小生成树（MST）
- 保持所有节点连通所需的最少边集合，优先保留高权重边。
- **直觉**：如果图中有 A→B(0.90) 和 B→C(0.90)，则 A→C(0.80) 是冗余的——MST 会删掉它，因为已有更好的路径连通 A 和 C。
- **用途**：找出知识体系的"骨架"——哪些链接是骨干，哪些是锦上添花。

### 图直径
- 最远两个节点的最短路径长度。全局"知识宽广度"指标。

## 链接预测

用于发现"应该连但没连"的笔记对：

| 方法 | 公式 | 直觉 |
|:---|:---|:---|
| 共同邻居 | $\|N(u) \cap N(v)\|$ | 共同朋友越多，越该认识 |
| Jaccard 系数 | $\frac{\|N(u) \cap N(v)\|}{\|N(u) \cup N(v)\|}$ | 归一化版，避免大度节点霸榜 |
| Adamic-Adar | $\sum_{w \in N(u)\cap N(v)} \frac{1}{\log\|N(w)\|}$ | 稀有的共同朋友比谁都认识的那个更有信息量 |

Adamic-Adar 更可靠——它惩罚"谁都认识"的超级枢纽（如被删除的 MOC），奖励小众的共同邻居。

## 工作流

1. 修改笔记 → 跑 `knowledge_graph_analyzer.py` 更新图
2. 检查幽灵链接数（必须为 0）
3. 用检索器命令导航（替代文件夹）
4. 对高 Jaccard/Adamic-Adar 分的笔记对评估是否需要加链接
5. 对新接入的素材笔记按权重规则建立单向链接
